# NFToken — Full-Stack ERC-721 dApp

A production-grade ERC-721 NFT collection dApp built with **OpenZeppelin Contracts v5**, **Foundry**, **Next.js 16**, **Wagmi v2**, and **RainbowKit**.

## Smart Contract: `NFToken.sol`

Inherits OpenZeppelin v5 contracts:

- **`ERC721`** — Core ERC-721 token standard
- **`ERC721Enumerable`** — On-chain token enumeration (totalSupply, tokenByIndex, tokenOfOwnerByIndex)
- **`ERC721URIStorage`** — Per-token metadata URI storage
- **`Ownable`** — Role-based access control (only owner can mint)

### Key Functions

| Function | Description |
|----------|-------------|
| `mint(address to, string memory uri)` | Mint a new NFT (owner only). Returns `tokenId`. |
| `burn(uint256 tokenId)` | Burn a token. Caller must be owner or approved. |
| `totalSupply()` | Total minted tokens across all owners |
| `tokenByIndex(uint256 index)` | Global token ID by index for enumeration |
| `tokenOfOwnerByIndex(address owner, uint256 index)` | Token ID by owner's index |
| `balanceOf(address owner)` | Number of tokens owned |
| `ownerOf(uint256 tokenId)` | Owner of a specific token |
| `tokenURI(uint256 tokenId)` | Metadata URI for a token |

### Custom Errors

| Error | Description |
|-------|-------------|
| `NFToken__MintToZeroAddress()` | Reverts when minting to `address(0)` |
| `NFToken__TokenDoesNotExist(uint256 tokenId)` | Reverts when querying a non-existent token |

## Frontend Overview

The frontend is a single-page dApp built in `frontend/src/app/page.tsx` with modular components:

| Component | Description |
|-----------|-------------|
| **ConnectScreen** | Hero screen with logo, description, and Connect Wallet button. Shown when not connected. |
| **DashboardHeader** | App bar with NFToken logo, chain indicator, and RainbowKit's `<ConnectButton />` |
| **StatsBar** | 4-card grid: Total Supply, Your Balance, Network, Account (truncated) |
| **MintTab** | NFT creation form: title, description, image upload (drag & drop), live preview, IPFS upload + transaction |
| **InventoryTab** | Grid of owned NFTs with token IDs, names, descriptions, and images from resolved metadata |
| **ToastContainer** | Fixed-position toast notifications (4s auto-dismiss) |

The app uses `useReadNfTokenTotalSupply` and `useWriteNfTokenMint` from auto-generated hooks in `generated.ts`. Inventory is fetched via the `useInventory` hook which calls `balanceOf` → `tokenOfOwnerByIndex` → `tokenURI` → metadata parsing.

### IPFS Provider Support

Two providers, configured via `NEXT_PUBLIC_IPFS_PROVIDER`:

| Provider | How It Works | Requires |
|----------|-------------|----------|
| `local` (default) | Uploads via Next.js API route → local IPFS Desktop daemon | IPFS Desktop running on `http://127.0.0.1:5001` |
| `pinata` | Direct upload to Pinata API | `NEXT_PUBLIC_PINATA_JWT` |

## Prerequisites

- [Foundry](https://book.getfoundry.sh/) — `forge`, `cast`, `anvil`
- [Node.js](https://nodejs.org/) v18+ & npm
- [IPFS Desktop](https://docs.ipfs.tech/install/ipfs-desktop/) (optional — for local IPFS uploads)
- [WalletConnect Project ID](https://cloud.walletconnect.com) (optional — without it, only injected wallets like MetaMask work)

## Quick Start

### 1. Install All Dependencies

```bash
make install-all
```

This runs `forge install` for OpenZeppelin contracts + forge-std, and `npm install` for the frontend.

### 2. Build Contracts & Generate ABI Hooks

```bash
make wagmi-gen
```

Compiles Solidity (forge build) and runs `npx wagmi generate` to produce fully typed React hooks in `frontend/src/generated.ts`.

### 3. Run Contract Tests

```bash
make contracts-test
```

Runs the full Foundry test suite (46 tests).

### 4. Run Frontend Tests

```bash
make frontend-test
```

## Local Deployment (Anvil)

### Step 1 — Start Anvil

```bash
# Terminal 1: Start a local Ethereum node
anvil
```

### Step 2 — Deploy the Contract

```bash
# Terminal 2: Deploy with interactive private key prompt
make deploy-anvil
```

### Step 3 — Start the Frontend

```bash
make frontend-dev
```
